Executive Summary

Multiple, concurrent threats elevate OT/ICS risk to critical. Actively exploited perimeter flaws (Check Point IKEv1 auth bypass; widespread Fortinet credential compromise) and nation‑state activity (PRC covert device networks; Iranian PLC intrusions; pro‑Russia VNC abuse) create credible, near‑term pathways into industrial environments. OT vendors released impactful advisories (Rockwell MNRF controller fault; Rockwell EN2/EN3 DoS; Siemens SICAM 8 issues; SALTO ProAccess escalation), while actively exploited enterprise vectors (Microsoft AD FS/SharePoint; WordPress “WP2Shell”; GeoServer RCE) increase pivot risk to OT. EV charging, building automation, and industrial data acquisition platforms show critical vulnerabilities in recent CISA bulletins.

Immediate actions: Disable IKEv1; rotate FortiGate creds; remove OT from internet Prioritize: Rockwell/Siemens patching; SharePoint/AD FS; WordPress Harden: VNC, VPN, identity, and DMZ boundaries

  • Rotate FortiGate admin/VPN/API creds; restrict admin plane to OOB; audit logs for config exports and rogue accounts; upgrade to FortiOS builds enforcing PBKDF2.
  • Patch Microsoft AD FS and SharePoint; enable AMSI Full Mode; hunt for webshells, machine key theft, and anomalous token grants.

OT Network Controls

  • Remove PLCs/HMIs from the internet; place OT behind an ICS DMZ with DPI firewalls; restrict CIP/Modbus/S7 ports (44818/2222/502/102).
  • Enforce controller RUN keyswitch; lock panels; require authenticated, least‑privilege downloads; monitor for program load attempts.
  • Patch Rockwell/Siemens advisories; validate backups and recovery images; maintain spares for critical controllers.

DMZ/Web and Cloud

  • Patch WordPress to fixed versions; deploy WAF rules; hunt for shells in wp‑content/cache and unauthorized admin users.
  • Patch/segregate GeoServer; restrict admin/UI exposure; disable xp_cmdshell on SQL Server; monitor BITS/PowerShell abuse.
  • Harden M365: restrict OAuth consent; monitor Graph calendar create/update with attachments and far‑future events; review service principals.

Device & Edge Hygiene

  • Patch edge devices (Ivanti, PAN‑OS, Cisco); audit tunnels, ACLs, SNMP/TACACS+/RADIUS; enforce SNMPv3 and CoPP; disable Smart Install.

  • Segment EVSE, CCTV, and BMS onto dedicated VLANs; enforce mutual TLS on OCPP; disable unused APIs (Hikvision ISAPI) and remote management.
