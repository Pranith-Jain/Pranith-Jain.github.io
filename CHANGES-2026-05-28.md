# New Features & Improvements — May 28, 2026

## 🆕 New CTI Providers (Free, Zero/Low Auth)

### 1. Shodan InternetDB

- **File**: `api/src/providers/shodan-internetdb.ts`
- **Auth**: ❌ None required
- **Rate Limit**: Generous (thousands/day)
- **Data**: Open ports, CVEs, hostnames, CPEs, tags
- **Scoring**: CVEs weighted by actively-exploited status, risky ports detected
- **Weight in composite**: 2 (for both IPv4/IPv6)

### 2. Spur.us

- **File**: `api/src/providers/spur.ts`
- **Auth**: ❌ None for community endpoint
- **Rate Limit**: Generous
- **Data**: VPN/proxy/tor/relay/hosting detection, service name
- **Scoring**: Tor highest (40), proxy (20), VPN (15), hosting (5)
- **Weight in composite**: 1

### 3. CrowdSec CTI

- **File**: `api/src/providers/crowdsec.ts`
- **Auth**: ✅ Free API key (1000 lookups/month)
- **Data**: Threat score, behaviors, attack details, community votes
- **Scoring**: Uses CrowdSec's own threat score (weighted composite)
- **Weight in composite**: 3
- **Env var**: `CROWDSEC_API_KEY`

### 4. IPinfo.io

- **File**: `api/src/providers/ipinfo.ts`
- **Auth**: Optional token (50k requests/month with free registration)
- **Data**: Geo, ASN, company, privacy detection, abuse contact
- **Scoring**: Privacy flags + hosted domains count
- **Weight in composite**: 1
- **Env var**: `IPINFO_TOKEN`

## 🚀 New Features

### 5. Threat Report Parser

- **File**: `api/src/routes/report-parser.ts`
- **Endpoint**: `POST /api/v1/report/parse`
- **MCP Tool**: `parse_threat_report`
- **Capabilities**:
  - Extract IOCs (IPs, domains, URLs, hashes) from unstructured text
  - Identify threat actors and malware families (50+ known actors, 30+ malware)
  - Extract MITRE ATT&CK techniques
  - Extract CVEs
  - AI-powered executive summary generation
  - Sector and product identification
- **AI Model**: Workers AI Llama 3.1 8B (free tier)
- **Input**: `{ text: "..." }` or `{ url: "https://..." }`

### 6. IOC Lifecycle Tracker

- **File**: `api/src/routes/ioc-lifecycle.ts`
- **Endpoints**:
  - `GET /api/v1/ioc-lifecycle?indicator=<ioc>` — lifecycle data for specific IOC
  - `GET /api/v1/ioc-lifecycle/trending` — most active IOCs in 24h
  - `GET /api/v1/ioc-lifecycle/stats` — aggregate statistics
- **MCP Tools**: `get_ioc_lifecycle`, `get_trending_iocs`
- **D1 Migration**: `migrations/0007_ioc_lifecycle.sql`
- **Capabilities**:
  - Track first/last seen timestamps
  - Monitor observation count and peak scores
  - Calculate decay rates (exponential moving average)
  - Activity status: active/declining/dormant/archived
  - Trend detection: rising/stable/declining

### 7. YARA Rule Generator

- **File**: `api/src/routes/yara-generator.ts`
- **Endpoints**:
  - `POST /api/v1/yara/generate` — generate YARA rule from description
  - `POST /api/v1/yara/validate` — validate YARA rule syntax
- **MCP Tools**: `generate_yara_rule`, `validate_yara_rule`
- **Capabilities**:
  - AI-powered rule generation (Workers AI)
  - Syntax validation with error reporting
  - MITRE ATT&CK technique mapping
  - Confidence scoring (syntax + detection)
  - Support for basic/standard/advanced complexity

### 8. Certificate Transparency Domain Monitor

- **File**: `api/src/routes/ct-monitor.ts`
- **Endpoints**:
  - `GET /api/v1/ct-monitor/watched` — list watched domains
  - `POST /api/v1/ct-monitor/watch` — add domain to watch
  - `DELETE /api/v1/ct-monitor/watch/:domain` — remove domain
  - `GET /api/v1/ct-monitor/certs?domain=...` — get certificates
- **MCP Tools**: `watch_domain_ct`, `get_domain_certs`
- **D1 Migration**: `migrations/0008_ct_monitor.sql`
- **Capabilities**:
  - Monitor CT logs via crt.sh (free, unlimited)
  - Alert on new subdomains
  - Detect suspicious domain patterns (typosquat, homograph)
  - Flag wildcard certificates
  - Detect short validity periods
  - Flag IP-based certificates
