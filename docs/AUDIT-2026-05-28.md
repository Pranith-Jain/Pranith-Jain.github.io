## CRITICAL Issues

### C1. SSRF via phishing-fingerprint fetchPageHandler
**File:** `api/src/routes/phishing-fingerprint.ts:20`
The `fetchPageHandler` calls `fetch(body.url)` directly on user-supplied input with zero SSRF protection. No `assertPublicHost`, no `pinnedFetch`, no allow-list. An attacker can probe internal IPs, cloud metadata endpoints (169.254.169.254), or the worker's own origin.
**Fix:** Use `pinnedFetch` or `assertPublicHost` like `url-preview.ts` and `web-scan.ts` do.

### C2. feeds-aggregate exceeds 50-subrequest limit on cold cache
**File:** `api/src/routes/feeds-aggregate.ts`
`MAX_FEEDS = 80` allows 80 URLs, but the Worker subrequest limit is 50. A novel 80-URL query silently drops 30+ feeds.
**Fix:** Reduce `MAX_FEEDS` to 45 or implement client-side batching.

### C3. Briefing self-heal blocks ALL cache warming
**File:** `worker/index.ts:737-796`
When `rebuiltThisHour = true`, the entire hourly warm cycle is skipped. A persistently failing briefing (e.g., NVD down) means ALL caches go cold.
**Fix:** Decouple briefing rebuild from warm cycle — run in separate `ctx.waitUntil` calls.

## HIGH Issues

### H1. pageviews bypasses rate limiting + does D1 writes
**Files:** `api/src/lib/ratelimit.ts:73`, `api/src/routes/pageviews.ts:40`
`/api/v1/pageviews` is in `BYPASS_EXACT`. The `?inc=1` path does a D1 write on every request. Completely unthrottled — attacker can burn D1 write quota freely.
**Fix:** Remove from bypass set or add a write-specific rate limit.

### H2. ~18 routes leak raw err.message in production
Routes: copilot, automation, malpedia, maltrail, maltiverse, inquest, certspotter, cloudflare-radar, hackertarget, triage, phishing-fingerprint, leakix, abuse-rss, campaign-generator, actor-enrich-stream, feeds-aggregate, ioc.
Error messages reveal upstream hostnames, API key presence, DNS failures, internal service names.
**Fix:** Wrap all `e.message` in `safeErrorMessage()` or return generic error strings.

### H3. No global authenticate(true) — all non-admin routes public
**File:** `api/src/index.ts:156`
`authenticate(false)` means every route is accessible without authentication. API key system exists but is entirely optional.
**Decision:** This may be intentional for a public portfolio — document the decision.

### H4. phishing-fingerprint has no body size limit
**File:** `api/src/routes/phishing-fingerprint.ts:17`
No body size limit, no URL validation, returns raw upstream HTML (XSS vector if rendered).
**Fix:** Add `safeJsonBody()` middleware, validate URL protocol, limit response size.

### H5. watches.ts — no body size limit, weak webhook URL validation
**File:** `api/src/routes/watches.ts:16-48`
`body.value` (IOC/keyword/actor) is unbounded length. Webhook URL accepts `javascript:` and `file://` protocols.
**Fix:** Add length limits, restrict webhook to `https://` only.

### H6. dashboard.ts — no input validation on watchlist entries
**File:** `api/src/routes/dashboard.ts:49-61`
Arbitrary strings persisted to KV and used in cache key lookups.
**Fix:** Validate domains/emails format before persisting.

### H7. phishing-fingerprint KV keys never expire
**File:** `api/src/routes/phishing-fingerprint.ts:41`
`expirationTtl: 0` means keys accumulate forever — unbounded KV growth.
**Fix:** Set TTL to 30 days.

### H8. Admin-purge leaks CF API error details
**File:** `api/src/routes/admin-purge.ts:78`
Returns raw Cloudflare API error body which can contain zone IDs, token permission details.
**Fix:** Return generic error message.

## MEDIUM Issues

### M1. Math.random() for API key generation
**File:** `api/src/lib/auth.ts:110`
Not cryptographically secure. Should use `crypto.getRandomValues()`.

### M2. CSRF guard bypassed when no Origin/Referer
**File:** `api/src/lib/csrf-guard.ts:72-74`
Non-browser clients (curl, scripts) skip CSRF protection entirely.

### M3. ioc-correlation not in hourly cron warm list
18-feed fan-out that's only visitor-warmed. First visitor after cold start pays full cost.
**Fix:** Add to `perSourceTargets` in the cron handler.

### M4. Most external APIs lack retry logic
Only NVD, EPSS, ip-geo, Wayback use `fetchResilient`. abuse.ch, Telegram, Reddit, Bluesky all do single-attempt fetches.
**Fix:** Adopt `fetchResilient` with `attempts: 2` for high-traffic APIs.

### M5. Cron warm budget is tight
Intel-bundle warmer burns ~37 of 50 subrequest budget. Publisher + telegram archive + warm targets total ~61 subrequests.
**Fix:** Move intel-bundle warmer to its own cron trigger.

### M6. feeds-aggregate merged cache TTL too short (1 min)
Per-URL cache is 1h. Merged response could safely be 5 min.

### M7. No API key rotation monitoring
24 API keys with no expiry checks or rotation schedule. A stale VT_API_KEY silently degrades IOC checker.

### M8. Inconsistent error response shapes
Most routes return `{ error: string }` but some include `message`, `detail`, or `generated_at` alongside errors.
**Fix:** Standardize on `{ error: string, message?: string }`.

### M9. 17 frontend fetch() calls missing AbortController
Pages: FeedStatus, MaltrailTrails, CampaignDetail, VictimReleaks, MyThreatIntel, ActorKb, Signal, XFirehose, RedditFirehose, TelegramSettings, Negotiations, RelationshipGraph, Blog, BlogPost, Briefings, DetectionLab, YaraManager.
**Fix:** Add `AbortController` pattern to each useEffect fetch.

### M10. 30+ `as unknown as` type assertions
Across: RelationshipGraph, StixGraph, FullSpectrum, SecurityGroupAnalyzer, ExifParse, DetectionLab, StixViewer, AzureRbacAnalyzer, Infostealer, XWatch, YaraManager, searchable-content, useDataFetch.
**Fix:** Define proper API response interfaces.

### M11. Briefings admin uses separate token
**File:** `api/src/routes/briefings.ts:159-170`
Uses `BRIEFINGS_ADMIN_TOKEN` instead of shared `ADMIN_TOKEN`. Two tokens to rotate.

### M12. campaigns.ts returns 200 with error field
**File:** `api/src/routes/campaigns.ts:151`
Returns 200 with `error: 'KV not configured'` when KV is missing.
**Fix:** Return 503.

### M13. Missing Cache-Control headers on watches and dashboard responses
**Files:** `api/src/routes/watches.ts`, `api/src/routes/dashboard.ts`
**Fix:** Add `Cache-Control: no-store` to mutation responses.

### M14. intel-bundle.ts bypasses Hono middleware chain
**File:** `api/src/routes/intel-bundle.ts:212-217`
Uses raw `new Response()` instead of `c.json()`. Error handler won't catch errors.

## Positive Findings

1. **All external fetches have timeouts** — no unbounded fetch calls found
2. **Lazy loading on all ~120 page components** — excellent code splitting
3. **Strong a11y foundation** — 237+ aria-labels, role="alert" on errors, SkipToContent, aria-busy
4. **Consistent useEffect cleanup** — `cancelled` boolean and AbortController patterns in ~50+ effects
5. **CORS locked to production origin** — no wildcard CORS
6. **Admin mutation rate limiting is extra-strict** — 5 req/min on top of global 30/min
7. **No hardcoded secrets** — all typed as optional with no defaults
8. **No `: any` type annotations** in production code
9. **KV key naming is collision-resistant** — well-separated prefixes
10. **Cache writes use `executionCtx.waitUntil`** — non-blocking pattern
11. **No dangerous race conditions** — worst case is redundant computation
12. **Cron dedup via KV locks** — prevents double-fire

## Enhancement Opportunities

### E1. Add EPSS scores to CVE pipeline
The CVE recent cache has NVD data but no EPSS scores. EPSS predicts exploitation likelihood — the single most important field for CVE triage after CVSS.

### E2. Add Maltrail data to copilot/unified-search
Maltrail cache exists but is not consumed by copilot or unified-search. Provides actor-to-IOC mapping.

### E3. Add Negotiations data to unified-search
Negotiations cache exists but unified-search doesn't include it. Unique ransom financial data.

### E4. Add Breach Forums / Stealer Intel to copilot
Both caches exist but are not consumed by copilot. Dark web monitoring data.

### E5. Add Onion Watch to copilot
Onion watch cache exists but copilot doesn't read it. Tor infrastructure data.

### E6. Add Phishing URLs to copilot
Phishing URL cache exists but copilot doesn't read it. Domain phishing campaign data.

### E7. Unified search should include all copilot sources
Copilot now reads 13 sources directly. Unified search reads 12 but misses negotiations. Align them.

### E8. Add cron warming for ioc-correlation, writeups, cybercrime, negotiations, breach-forums
These are visitor-driven only. Adding them to the hourly cron eliminates cold-start penalty.

### E9. Add circuit breaker dashboard
Circuit breaker state is per-colo in-memory. A dashboard showing circuit status per provider would improve observability.

### E10. Add API key health monitoring
A feed-status widget showing which API keys are valid/expired would prevent silent degradation.

### E11. Migrate pages to useDataFetch hook
17 pages have ad-hoc fetch patterns. The existing `useDataFetch<T>` hook provides typed responses, AbortController, error states, and retry — migrate pages to use it.

### E12. Standardize error responses
Create a shared `errorResponse(c, status, code, message)` helper. All routes should return `{ error: string, message: string }`.
