# Tailwind → Panda CSS Migration Log

Tracks the phased replacement of Tailwind CSS with Panda CSS in this
portfolio. The goal is zero visual change while gaining typed recipes,
build-time extraction, and a real design-system source of truth.

## Status

- [x] **Phase 0** — Install Panda, configure build, set up tokens & recipes
- [ ] **Phase 1** — Migrate design-system primitives (`Surface`, `Card`, `SeverityPill`, `Button`)
- [ ] **Phase 2** — Migrate shared/leaf components (SeverityPill consumers, `Card`, `Badge`)
- [ ] **Phase 3** — Migrate one low-risk route (e.g. `About.tsx`)
- [ ] **Phase 4** — Migrate remaining routes page-by-page
- [ ] **Phase 5** — Delete Tailwind, remove PostCSS plugin, drop `tailwind.config.js`

## Architecture: Dual-Pipeline

Phases 0-4 run Tailwind and Panda side-by-side. Tailwind classes keep
working everywhere; Panda recipes get added incrementally. The build
emits both `dist/assets/index-*.css` (Tailwind) and injects
`styled-system/styles.css` (Panda) via a `?raw` import in `main.tsx`.

**Why the `?raw` import?** Tailwind v3's `normalizeTailwindDirectives`
PostCSS plugin fails on Panda's `@layer base` blocks (it expects a
matching `@tailwind base` directive in the same file). The `?raw`
query forces Vite to return Panda's CSS as a string, bypassing the
PostCSS pipeline. The CSS is injected at runtime via a `<style>` tag.

## How to Use a Recipe

```tsx
import { card, severityPill } from '@/styled/recipes';

<div className={card({ variant: 'card', interactive: true, size: 'md' })}>
  <span className={severityPill({ tone: 'critical' })}>Critical</span>
</div>;
```

The `className` returned is a hashed atomic class like
`c-jHfudj c-iyVnGC`. The CSS comes from `styled-system/styles.css`.

## Commands

- `npx panda codegen` — Generate `styled-system/` from recipe files
- `npx panda codegen --watch` — Watch mode for dev
- `npx panda extract` — Force CSS extraction (used in `prebuild`)

| ------------------------------------------------- | -------------------------------------- |
| `bg-brand-500`                                    | `bg: 'brand.500'`                      |
| `text-muted`                                      | `color: 'muted'`                       |
| `shadow-e2`                                       | `boxShadow: 'e2'`                      |
| `text-meta`                                       | `fontSize: 'meta', lineHeight: 'meta'` |
| `rounded-xl`                                      | `borderRadius: 'xl'`                   |
| `surface-card`                                    | `card({ variant: 'card' })`            |
| `bg-rose-500/15 text-rose-700 dark:text-rose-300` | `severityPill({ tone: 'critical' })`   |

## Migration Gotchas

1. **Panda's static analysis is lazy.** Recipes are only emitted as
   CSS when at least one call site invokes the recipe function. If
   you add a recipe and don't see its CSS, you haven't used it yet.
2. **Tree-shaking eats the `?raw` import** if the variable isn't
   observably used. We anchor it to `globalThis.__PANDA_CSS_INJECTED__`
   in `main.tsx` so Rollup can't eliminate it.
3. **Panda's reset overlaps with Tailwind's.** Some elements get
   double-reset (e.g. `*, *::before, *::after { margin: 0 }` runs
   twice). During the dual-pipeline this is harmless because both
   resets are idempotent. Phase 5 will deduplicate.
4. **Panda doesn't support the `@apply` directive.** Use `css({...})`
   template literals or compose with `cx()`.

## Phase 5 — Tailwind removal (final)

**Date:** 2026-06-21
**Status:** ✅ Complete. Panda CSS is the sole styling system.

### What changed

1. **`package.json`** — removed `tailwindcss`, `tailwindcss-animate`, `tailwind-merge`, `@tailwindcss/typography` from `devDependencies` (35 transitive packages removed from `node_modules`).
2. **`postcss.config.js`** — dropped the `tailwindcss` + `autoprefixer` Tailwind chain. New chain is:
   - `@pandacss/dev/postcss` — handles Panda's `@layer base` blocks and emits the compiled token CSS
   - `autoprefixer` — vendor-prefixes Panda's output
3. **`tailwind.config.js`** — deleted. `panda.config.ts` is now the single source of design tokens.
4. **`src/main.tsx`** — replaced the `?raw` runtime Panda CSS injection (a Phase 0–4 workaround for Tailwind v3's PostCSS plugin failing on Panda's `@layer` blocks) with a canonical direct import:
   ```ts
   import './index.css';
   import '../styled-system/styles.css';
   ```
5. **`src/index.css`** — stripped all Tailwind directives:
   - 3× `@tailwind base|components|utilities` removed
   - 3× `theme('colors.xxx')` JS-style color references → literal hex values
   - 13× `@apply ...` directives → plain CSS using literal hex + existing CSS variables (`--surface-*`, `--border-*`, `--muted`)
6. **`scripts/check-budgets.mjs`** — `index-*.js` budget restored to 320KB / 105KB (post-migration bundle fits comfortably under the relaxed Phase 0–4 budget).
7. **Source comments** — 4 stale "tailwind" comments updated in `CaseStudy.tsx`, `F3ead.tsx`, `SocVulns.tsx`, `AgentMap.tsx`.
8. **`package.json` keywords** — `tailwind` → `panda-css`.
9. **`scripts/check-no-px-text.mjs`** — docstring updated. Script still finds dead Tailwind ad-hoc strings in `InfraMap` / `Projects` / `McpStatusBanner` (exit 0, expected; not blocking).
10. **Panda peer dep** — installed `postcss-selector-parser` (missing peer of `@pandacss/dev`; required by `panda codegen`).
